\section{客户端-服务器编程模型}
{
    每个网络应用都是基于\emreg{客户端-服务器模型}的。
    采用这个模型，一个应用是由一个\emreg{服务器}进程和一个或多个\emreg{客户端}进程组成。
    服务器管理某种\emreg{资源}，并且通过操作这种资源来为它的客户端提供某种\emreg{服务}。

    客户端-服务器模型中的基本操作是\emreg{事务(transaction)}。

    \begin{enumerate}
        \item 当一个客户端需要服务时，它向服务器发送一个\emspe{请求}。
        \item 服务器收到请求后，解释它，并以适当的方式操作它的资源。
        \item 服务器给客户端发送一个\emspe{响应}，并等待下一个请求。
        \item 客户端收到响应并处理它。
    \end{enumerate}

    客户端和服务器时进程，而不是常提到的机器或者\emreg{主机}。
}
