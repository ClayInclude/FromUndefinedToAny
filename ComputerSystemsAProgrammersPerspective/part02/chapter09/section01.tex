\section{物理和虚存地址}
{
    计算机系统的主存被组织成一个由M个连续的字节大小的单元组成的数组。
    每个字节都有一个唯一的\emreg{物理地址(Physical Address, PA)}。
    给定这种简单的结构，CPU访问内存的最自然的方式就是使用物理地址。
    这种方式称为\emreg{物理寻址(physical addressing)}。

    现代处理器使用的是一种称为\emreg{虚拟寻址(virtual addressing)}的寻址形式。

    使用虚拟寻址，CPU通过生成一个\emreg{Virtual Address, VA}来访问主存，这个虚拟地址在被送到内存之前先转换成适当的物理地址。
    将一个虚拟地址转换为物理地址的任务叫做\emreg{地址翻译(address translation)}。
    CPU芯片上叫做\emreg{内存管理单元(Memory Management Unit, MMU)}的专用硬件，利用存放在主存中的查询表来动态翻译虚拟地址，该表的内容由操作系统管理。
}
